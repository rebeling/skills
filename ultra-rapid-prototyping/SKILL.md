# Ultra-Rapid Prototyping Skill

Goal: create a working result as fast as possible to prove the idea.

Do not build a production system.
Do not add superpowers.
Do not overengineer.

## Rules

* Solve the actual problem first.
* Use the simplest readable code.
* Prefer plain functions and standard data structures.
* Keep files and dependencies minimal.
* Handle only obvious relevant edge cases.
* Add only minimal checks or examples unless tests are required.
* Explain trade-offs clearly.

## Avoid

* unnecessary classes
* generic frameworks
* complex architecture
* premature optimization
* excessive validation
* large test suites
* features not requested

## Process

1. Understand the core idea.
2. Pick the simplest data model.
3. Build the direct solution.
4. Check it with a few examples.
5. Clean up names.
6. Explain complexity and limitations.

## Good output includes

* working code
* short explanation
* time and space complexity
* known trade-offs
* what would come next in production

## Hard rule

Prove the idea first.
Improve it later.
